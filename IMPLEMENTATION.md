# Implementation Summary

## Features Implemented

For Project Milestone 4, our team worked on implementing core components of the Canvas Deadline Tracker. The goal of this milestone was to begin translating our design from PM3 into actual implementation.

During this milestone, we focused on building the basic structure of the system and implementing several core features. These included deadline tracking, assignment organization, notification support, and filtering functionality. We also worked on data modeling and local state management to support storing assignment and course information.

These implementations help move our project toward the main goal of helping students track deadlines across multiple Canvas courses more easily.

## AI Tools Used

During implementation, our team used AI tools including ChatGPT and GitHub Copilot. These tools were used mainly for brainstorming ideas, generating initial code structure, and helping us move forward when we were unsure how to implement certain features.

## How AI Was Used

AI was used to assist with:

- generating initial function structures
- suggesting implementation approaches
- improving code organization
- helping with debugging and logic clarification
- supporting documentation writing

AI suggestions helped us speed up development and provided helpful starting points for implementation.

## Generated Code Behavior

The AI-generated code and suggestions mainly helped with building structure and implementing logic related to assignment tracking, filtering, notifications, and dashboard functionality. These suggestions were used as a starting point and were reviewed and adjusted before being added to the repository.

## Modifications Made

After using AI-generated suggestions, we made several modifications, including:

- adjusting logic to match project requirements
- updating naming conventions
- improving code readability
- integrating generated code with existing modules
- fixing issues during testing and integration

These changes were necessary to ensure the code fit our overall system design.

## Did It Work as Expected

AI suggestions were helpful but did not always fully match our project needs. Some generated code worked immediately, while other parts required changes before being integrated into the system. Overall, AI helped speed up the development process.

## Why AI Was Used

We used AI tools to help generate ideas, support implementation, and improve productivity. AI helped us move forward more quickly, especially when working on unfamiliar parts of the system. However, all generated content was reviewed and modified before being included in the project.

## Reflection

Using AI tools helped our team begin implementation more efficiently and provided useful ideas during development. We still needed to review and modify generated code to ensure it matched our project design and requirements. Overall, AI supported our development process while we maintained control over final implementation decisions.
