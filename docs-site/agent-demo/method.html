This page documents the full training methodology: SFT trajectory collection, DPO preference optimization, the 9-method ablation matrix, pre-execution guardrails, and the bench harness used to validate results.

The Base Model

Gemma-4-E2B-IT base

Google's MatFormer architecture — a nested transformer where the same weights serve multiple effective parameter counts. At the 2B extraction point the model is multimodal (text + vision tokens) and instruction-tuned. Fine-tuning inherits the instruction-following alignment without catastrophic forgetting of general capabilities.

Supervised Fine-Tuning (SFT)

Phase 1 — teaches the model what a correct tool call looks like

SFT trains on 400–500 human-curated tool-call trajectories. Each trajectory is a full user-turn → model-turn sequence where the model outputs one or more native Gemma-4 tool calls followed by a synthesized answer. The model learns correct syntax, correct tool selection, and correct argument structure from direct imitation.

Native Gemma-4 tool-call format

<|tool_call>call:canvas.get_assignments{course_id: "CS3704", days_ahead: 7}<tool_call|>

The <|tool_call> / <tool_call|> sentinel tokens are part of Gemma-4's native vocabulary. The parser mirrors the Python SDK's tool_parser.py exactly — same regex, same round-trip guarantee.

Direct Preference Optimization (DPO)

Phase 2 — teaches the model what a better tool call looks like

DPO (Rafailov et al. 2023, arXiv:2305.18290) replaces the RLHF reward model with a closed-form loss derived directly from the Bradley-Terry preference model. Given a preference pair (chosen, rejected), DPO increases the log-probability of the chosen completion relative to the rejected one, weighted by how much the current policy has already diverged from the reference.

Reference model role

The reference model is the SFT checkpoint — it acts as a KL-divergence anchor, not a teacher. The DPO objective penalizes moving too far from the SFT distribution, preventing mode collapse on easy preferences. With β = 0.3 (stronger regularization), this is critical for small-N datasets (< 2000 pairs) where overfitting is the primary failure mode.

Four preference pair types

parsimony

Chosen: 1 minimal tool call
Rejected: 4+ chained calls for same outcome
Trains the model to prefer the shortest correct solution.

wrong_tool

Chosen: correct tool name from SDK registry
Rejected: hallucinated tool name
Trains the model to stay within the 18-tool manifest.

wrong_args

Chosen: correct, complete argument set
Rejected: missing or semantically wrong args
Trains argument fidelity independently of tool selection.

format

Chosen: native <|tool_call> syntax
Rejected: natural-language description of a tool call
Reinforces structured output over prose fallback.

Pre-execution Guardrails (G1–G13)

Automated fail-stop CLI checks that run before every training job

Each guardrail encodes a specific failure mode discovered during prior training runs. Every check is deterministic and exits non-zero on violation — no training job starts if any guardrail fails. canvas-train --check runs all 13 in sequence.

G1 Tool-call token coverage

≥80% of chosen/rejected completions must contain the <|tool_call> sentinel. Catches data pipelines that drop the token.

G2 Eval dataset required

eval_dataset must exist and be item-disjoint from train. Prevents eval contamination and silent overfitting.

G3 Base model bench sanity

Quick bench against base model must show ≥20% first-tool accuracy. If base is already broken, no fine-tune will fix it.

G4 Tool distribution CV ≤ 0.3

Coefficient of variation across 18 tools must be ≤ 0.3, with min 30 examples per tool. Prevents distribution skew.

G5 Preflight grad norm

1-step dry run: grad_norm must be < 1.0. Catches numerical instability (NaN/Inf in activations) before a full run wastes GPU hours.

G6 Pipeline shrinkage ≤ 30%

Each processing stage (tokenize → filter → pair) must retain ≥ 70% of its input. Excessive shrinkage signals a pipeline bug, not a data quality issue.

G7 Tool name registry check

Every tool name in the dataset must appear in the SDK tool registry. Catches stale or renamed tool references before they poison the model.

G8 Serializer round-trip

Parser → serializer → parser round-trip must pass 100% of samples. A round-trip failure means the model will emit unparseable output.

G9 Behavioral reward margins

Micro-bench verifies reward margins move in the correct direction per pair type — parsimony pairs should widen the parsimony margin, not the format margin.

G10 Docker container smoke test

canvas-train --help inside the training container must exit 0. Catches broken dependencies and mis-built images before a multi-hour run.

G11 Sequence length budget

<1% of rows may exceed max_length=2048. Rows above the limit are silently truncated during training — high truncation rates indicate dataset construction errors.

G12 Minimum dataset sizes

DPO/KTO methods require ≥ 1000 preference pairs; SFT requires ≥ 200 trajectories. Below these thresholds overfitting is near-certain.

G13 Trajectory tool repetition

No single tool may appear more than 3× in one trajectory. Excessive repetition teaches the model to loop rather than complete — a failure mode we observed directly in pre-v7 runs.

Metrics

  • →Tool selection accuracy — did the model call the correct tool as its first action? Measured per tool, per method.
  • →Argument accuracy — ToolBench-style partial match on required arguments (exact for IDs, fuzzy for natural-language fields).
  • →Wilson 95% CI — each per-tool accuracy estimate comes with a Wilson score confidence interval. Small per-tool samples (<40) are flagged as low-confidence.
  • →Parsimony rate — fraction of correct responses that use the minimum-cardinality tool set. DPO should improve this vs SFT baseline.

Checkpoint selection

The production checkpoint is the epoch that maximizes macro-average tool selection accuracy on the eval split, subject to a constraint that no individual tool falls below 40% (tool-floor constraint). This prevents selecting a checkpoint that is strong on common tools but has regressed on rare ones.
