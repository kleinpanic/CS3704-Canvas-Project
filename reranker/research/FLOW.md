
## Quick Start (Path A — Try Now)

### 1. Copy data to spark-maker
```bash
scp ~/codeWS/Gemma2B-Reranker/data/rerank_sft.jsonl spark:/srv/spark-maker/datasets/
```

### 2. Run QLoRA training
```bash
ssh spark
cd /srv/spark-maker

docker compose run --rm trainer \
    bash /workspace/scripts/finetune-lora.sh \
    google/gemma-2b-it \
    /workspace/datasets/rerank_sft.jsonl \
    gemma2b-reranker \
    qlora
```

**Output**: `/srv/spark-maker/output/loras/gemma2b-reranker/`

### 3. Benchmark
```bash
docker compose run --rm trainer \
    python3 /workspace/scripts/benchmark.py \
    --adapter /srv/spark-maker/output/loras/gemma2b-reranker \
    --test /srv/spark-maker/datasets/rerank_test.jsonl \
    --output /srv/spark-maker/output/benchmarks/results.json
```

### 4. Export GGUF
```bash
docker compose run --rm converter \
    bash /workspace/scripts/convert-gguf.sh \
    /srv/spark-maker/output/loras/gemma2b-reranker \
    Q4_K_M
```

### 5. Copy results home
```bash
scp -r spark:/srv/spark-maker/output/loras/gemma2b-reranker ~/codeWS/Gemma2B-Reranker/outputs/
```

## Path B: DPO Distillation (After Path A)

### Prerequisites
```bash
# Stop Nemotron (frees 58GB)
forge unload 0

# Load teacher model
forge load nvidia/Gemma-4-31B-IT-NVFP4 --slot 0
```

### Step 1: Generate teacher preferences
```bash
# Use Gemma-4-31B via forge chat or API to label pairs:
python3 scripts/generate_teacher_preferences.py \
    --input /srv/spark-maker/datasets/rerank_clean.jsonl \
    --output /srv/spark-maker/datasets/rerank_dpo.jsonl \
    --teacher nvidia/Gemma-4-31B-IT-NVFP4
```

### Step 2: DPO train student
```bash
cd /srv/spark-maker

# Generate DPO config
python3 scripts/train_config_gen.py \
    --model google/gemma-2b-it \
    --method dpo \
    --dataset /srv/spark-maker/datasets/rerank_dpo.jsonl \
    --output /srv/spark-maker/configs/generated/gemma2b-dpo.yaml

# Train
docker compose run --rm trainer \
    python3 /workspace/LLaMA-Factory/src/train.py \
    /srv/spark-maker/configs/generated/gemma2b-dpo.yaml
```

## Gemma-4-31B Teacher Notes

**Model**: `nvidia/Gemma-4-31B-IT-NVFP4`
**Size**: 18GB (NVFP4), 31B params
**Context**: 256K
**Status**: On disk at `/srv/spark-ai-v2/state/hf-cache/hub/models--nvidia--Gemma-4-31B-IT-NVFP4/`
**Access**: `forge load nvidia/Gemma-4-31B-IT-NVFP4 --slot 0`

**Why it helps as teacher**:
- Stronger reasoning about urgency/priority
- Can explain WHY an item is more urgent (generates natural language reasoning)
- DPO training on teacher explanations → student learns urgency *patterns*, not just surface features
- Better cross-course generalization (learns general priority principles)

**Limitation**: Teacher knowledge is general web-trained; Canvas urgency is domain-specific. Best gains on hard negatives where heuristic is uncertain.
