## Architecture: Two Paths

### Path A — Direct QLoRA (Simple, Do First)

```
Gemma 2B IT (student) ──QLoRA──► LoRA adapter
Heuristic preferences ──────────► Training signal
```

**What it does**: Fine-tune Gemma 2B IT to match heuristic preference labels.
**Time on GB10**: ~20-40 min (GB10 integrated GPU, ~5-10 steps/sec for 2B)
**Memory**: ~2-4GB (fits alongside Nemotron with small batch)

### Path B — Offline DPO Distillation (Better, Do After A)

```
Step 1: Gemma-4-31B IT (teacher)
         │
         │ 1. Load nvidia/Gemma-4-31B-IT-NVFP4 into vLLM
         │ 2. For each pair: ask "which is more urgent AND WHY"
         │ 3. Extract teacher reasoning → "chosen"/"rejected" explanations
         ▼
Step 2: Synthetic DPO dataset
         { "prompt": query, "chosen": reason_A, "rejected": reason_B }
         (generated by Gemma-4-31B, not the heuristic)
         │
         ▼
Step 3: Gemma 2B IT (student) ──DPO──► LoRA adapter
```

**What it does**: Teach Gemma 2B to reason about urgency, not just memorize heuristic patterns.
**Advantage**: Learns *why* one item is more urgent, generalizes better to novel courses/types.
**Teacher cost**: ~10-20 min to generate 2000 preference explanations (31B NVFP4 in vLLM).
**Student training**: Same as Path A, just DPO format instead of SFT.
**Memory**: Teacher (18GB vLLM) + Student training (2-4GB) = ~22GB — need to stop Nemotron first.

## Dataset Format for LLaMA-Factory

**For SFT / QLoRA (Path A)**:
```jsonl
{"text": "[Query]: ...\nItem A: ...\nItem B: ...\nWhich is more urgent? Item A is more urgent.\nReason: ...<eos>"}
```

**For DPO (Path B)**:
```jsonl
{"prompt": "Which item is more urgent and why?\n[Query]: {q}\nItem A: {a}\nItem B: {b}", "chosen": "Item A is more urgent because: {reason_from_gemma4}", "rejected": "Item B is more urgent because: {reason}"}
```

The `collect_rerank_dataset.py export-sft` already produces the SFT format. Need a new `export-dpo` command for Path B.
